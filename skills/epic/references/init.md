# epic init

First-time repository setup for the epic workflow system.

## When to Run

- First time using the epic skill in a new repo
- After cloning a repo that doesn't have `.ai-queue/` set up
- To reconfigure project board integration

## Procedure

### 1. Auto-detect owner and repo

```bash
git remote get-url origin
```

Parse the URL to extract `OWNER` and `REPO`:
- `https://github.com/OWNER/REPO.git` → extract directly
- `git@github.com:OWNER/REPO.git` → extract directly
- Strip any trailing `.git` suffix

### 2. Check for existing setup

If `.ai-epics/docs/project-setup.md` already exists, print its contents and ask: **"Project already initialized. Re-run init?"** On "no", abort. On "yes", continue (overwrite).

### 3. Create directory structure

```bash
mkdir -p .ai-queue/_completed
mkdir -p .ai-epics/docs
mkdir -p .ai-epics/roadmaps
mkdir -p .ai-epics/archive
```

> **Note:** `.ai-queue/` is gitignored (per-engineer ephemeral work). `.ai-epics/` is tracked in git (shared project artifacts: docs, roadmaps, archive).

### 3.5. Create feature flag module

Create the flag directory and seed the TypeScript flag registry:

```bash
mkdir -p config
```

Write `config/flags.ts` from the template in `references/feature-flags.md` (the empty registry version). If this file already exists (project already uses feature flags), skip this step.

### 4. Create GitHub labels

Use `--force` for idempotency (safe to re-run):

```bash
gh label create "epic" --description "Multi-stage initiative" --color "8B5CF6" --repo OWNER/REPO --force
gh label create "stage" --description "Individual stage within an epic" --color "3B82F6" --repo OWNER/REPO --force
gh label create "iteration" --description "Fix from ITERATE phase" --color "F59E0B" --repo OWNER/REPO --force
gh label create "task" --description "Sub-issue: atomic unit of work within a stage" --color "10B981" --repo OWNER/REPO --force
```

### 5. Attempt project board setup

Check if the `project` scope is available:

```bash
gh auth status 2>&1
```

- **If `project` scope is available:**
  - List existing projects: `gh project list --owner OWNER --format json`
  - If a project exists, ask the user which project number to use (or skip)
  - If no project exists, create one: `gh project create --title "REPO Roadmap" --owner OWNER`
  - Capture `PROJECT_NUMBER`
- **If `project` scope is NOT available:**
  - Print: "GitHub `project` scope not available. Skipping project board setup. Run `gh auth refresh --hostname github.com -s read:project -s project` to enable later."
  - Set `PROJECT_NUMBER` to null (omit from setup file)

### 6. Generate `project-setup.md`

Write `.ai-epics/docs/project-setup.md` and commit it:

```markdown
---
owner: OWNER
repo: REPO
project_number: PROJECT_NUMBER
flags_dir: config
created: YYYY-MM-DD
---

# Project Setup

Generated by `epic init`. Provides project-specific values for the epic skill.

- **Repository:** OWNER/REPO
- **Project board:** https://github.com/orgs/OWNER/projects/PROJECT_NUMBER
- **Labels:** epic, stage, iteration
- **Feature flags:** config/flags.ts
```

If `PROJECT_NUMBER` is null, omit the `project_number` field and the project board line.

### 7. Install post-merge cleanup workflow

Create `.github/workflows/epic-cleanup.yml` to automate Phase 8 (roadmap archival, branch deletion) on PR merge:

```bash
mkdir -p .github/workflows
```

Write `.github/workflows/epic-cleanup.yml` with the following content:

```yaml
# Post-merge cleanup for the epic workflow.
# Archives the roadmap, commits the change, and deletes the epic branch.
name: Epic Post-Merge Cleanup

on:
  pull_request:
    types: [closed]
    branches: [main]

permissions:
  contents: write

jobs:
  cleanup:
    if: >-
      github.event.pull_request.merged == true && (
        startsWith(github.event.pull_request.head.ref, 'epic/') ||
        startsWith(github.event.pull_request.head.ref, 'feat/') ||
        startsWith(github.event.pull_request.head.ref, 'fix/') ||
        startsWith(github.event.pull_request.head.ref, 'chore/') ||
        startsWith(github.event.pull_request.head.ref, 'docs/')
      )
    runs-on: ubuntu-latest

    steps:
      - name: Checkout main
        uses: actions/checkout@v4
        with:
          ref: main
          fetch-depth: 0

      - name: Extract slug from branch name
        id: slug
        run: |
          BRANCH="${{ github.event.pull_request.head.ref }}"
          SLUG=$(echo "$BRANCH" | sed 's|^epic/||;s|^feat/||;s|^fix/||;s|^chore/||;s|^docs/||')
          echo "slug=$SLUG" >> "$GITHUB_OUTPUT"
          echo "branch=$BRANCH" >> "$GITHUB_OUTPUT"

      - name: Find roadmap file
        id: roadmap
        run: |
          # Look for a roadmap file whose name contains the slug
          MATCH=$(find .ai-epics/roadmaps -maxdepth 1 -name "*${{ steps.slug.outputs.slug }}*" -type f 2>/dev/null | head -1)
          if [ -z "$MATCH" ]; then
            echo "No roadmap file found for slug '${{ steps.slug.outputs.slug }}' — skipping."
            echo "found=false" >> "$GITHUB_OUTPUT"
          else
            echo "file=$MATCH" >> "$GITHUB_OUTPUT"
            echo "basename=$(basename "$MATCH")" >> "$GITHUB_OUTPUT"
            echo "found=true" >> "$GITHUB_OUTPUT"
          fi

      - name: Update roadmap status and move to _completed
        if: steps.roadmap.outputs.found == 'true'
        run: |
          FILE="${{ steps.roadmap.outputs.file }}"

          # Update status from in-review (or any status) to done
          sed -i 's/^\(- \*\*Status:\*\*\) .*/\1 done/' "$FILE"

          # Ensure _completed directory exists
          mkdir -p .ai-epics/archive

          # Move to archive
          mv "$FILE" ".ai-epics/archive/${{ steps.roadmap.outputs.basename }}"

      - name: Commit changes
        if: steps.roadmap.outputs.found == 'true'
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "41898282+github-actions[bot]@users.noreply.github.com"
          git add .ai-epics/
          git commit -m "chore(epic): archive roadmap for ${{ steps.slug.outputs.slug }}" || echo "Nothing to commit"
          git push

      - name: Delete epic branch
        run: |
          BRANCH="${{ steps.slug.outputs.branch }}"
          if git ls-remote --exit-code --heads origin "$BRANCH" > /dev/null 2>&1; then
            git push origin --delete "$BRANCH"
            echo "Deleted remote branch: $BRANCH"
          else
            echo "Branch $BRANCH already deleted — skipping."
          fi
```

### 8. Print summary

```
epic init complete.

Owner:   OWNER
Repo:    REPO
Project: #PROJECT_NUMBER (or "not configured")
Labels:  epic, stage, iteration, task
Flags:   config/flags.ts

Setup file: .ai-epics/docs/project-setup.md
```

## Notes

- This command is idempotent. Running it again overwrites `project-setup.md`.
- Labels use `--force` so they are safe to re-run.
- `.ai-queue/` is gitignored (per-engineer ephemeral task queue). `.ai-epics/` is committed to the repo for team visibility.
